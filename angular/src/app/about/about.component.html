API Overview

API is the acronym for Application Programming Interface, which is a remote server that our Javascript code can communicate with to usually pull in data from the database. Once the browser sends a request to the server, unless the internet is not functional, the response will always come back from the server. An API allows a user to sort through data in the most efficient way possible, as well as updating, adding, and deleting data, this makes it simple to pull only the information that we requested onto the webpage.

How APIs are used Everyday

APIs play a huge role in our everyday life since they allow us to easily share important data. API functions like a middleman (a virtual one, of course) as it relays information from one interface, like a website or mobile app.

  • Paying with PayPal
  • Streaming services (like Netflix and spotify (allowing user to search through different tpyes of music and movies ))
  • Using Google Maps in a ride share app
  • Logging to websites using other social sites
  • Weather Data
  • Twitter Bots
  • Embedding youtube videos into a webpage
  • Telsa (sending software updates via APIs)

Another good quality of an API is great error handling, it is important that an application provides the specific error code and human readable message in order to provide as much help to the developer so they can fix the issue. There are four basic types of APIs:

  • Private APIs are published internally for use by developers to improve its products and services. Private APIs are not exposed to third parties.
  • Public APIs are published publicly and can be used by any third party. There are no restrictions on these APIs.
  • Partner APIs can only be used by specific parties with whom the owner agrees to share data. Partner APIs are used within business relationships, often to integrate software between partnering companies.
  • Composite APIs combine multiple APIs to address related or interdependent tasks, and often improve speed and performance compared with individual APIs.

The make up of an API

So far we have learned that an API is how interfaces communicate with eachother, but how does that communication take place?

APIs consist of three parts:

  • User: the person executing the request
  • Client: the interface that sends the request to the server
  • Server: the interface that responds to the request

Actions You Can Take Through an API

Similar to this web application created, a server will be built first to hold the data. Once the server is running a developer can use the endpoints created by the programmer to filter or search the data on the server, or build thier own program that rus searches on the data base. If you havent realized it yet, you can build amazing things with APIs. However, before you jump right into to using APIs, it would be great to know the 4 types of actions an API can take:

  • GET: requests data from a server. This request can result in displaying all the data or specific to one single type of information depending on what the user would like to "GET" from the server.
  • POST: sends changes from the client to the server. This request is used to add new information to the server.
  • PUT: updates existing information. This request is used to revise or change any existing data on the server.
  • DELETE: deltes existing information. This request results in existing information on the server being deleted.
