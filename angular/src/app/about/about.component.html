API Overview

API is the acronym for Application Programming Interface, which is a remote server that our Javascript code can communicate with to usually pull in data from the database. Once the browser sends a request to the server, unless the internet is not functional, the response will always come back from the server. An API allows a user to sort through data in the most efficient way possible, as well as updating, adding, and deleting data, this makes it simple to pull only the information that we requested onto the webpage.

Identifying Good APIs

APIs are the first way developers access and interact with your products. A good API will enhance the user's experience, providing complete, accurate, and easy to read documentation. Another good quality of an API is great error handling, it is important that an application provides the specific error code and human readable message in order to provide as much help to the developer so they can fix the issue. Lastly, a great API is one that can be equally gentle with students in this program while empowering experienced users. There are four basic types of APIs:

  • Private APIs are published internally for use by developers to improve its products and services. Private APIs are not exposed to third parties.
  • Public APIs are published publicly and can be used by any third party. There are no restrictions on these APIs.
  • Partner APIs can only be used by specific parties with whom the owner agrees to share data. Partner APIs are used within business relationships, often to integrate software between partnering companies.
  • Composite APIs combine multiple APIs to address related or interdependent tasks, and often improve speed and performance compared with individual APIs.
